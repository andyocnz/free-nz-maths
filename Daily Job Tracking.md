## ‚ö†Ô∏è Critical Reminders

### Common Mistakes to Avoid

Based on today's fixes (Phase 10.12), ALWAYS remember:

#### 1. ‚ùå NEVER Hardcode Calculation Questions
```json
// ‚ùå BAD
{
  "stem": "Solve 2x + 5 = 15",
  "params": {},
  "answer": "5"  // Students memorize!
}

// ‚úÖ GOOD
{
  "stem": "Solve {a}x + {b} = {c}",
  "params": { "a": ["int", 1, 5], ... },
  "answer": "(c - b) / a"
}
```

#### 2. üé® visualData Must Match Params
```json
// ‚ùå BAD - Static visual with random params
{
  "params": {
    "scenario": ["choice", ["positive", "Positive"], ["negative", "Negative"]]
  },
  "visualData": { "points": [[1,40],[2,50],[3,60]] }  // Always upward!
}

// ‚úÖ GOOD - Reference params OR split templates
"visualData": { "points": [["x1","y1"], ["x2","y2"]] }  // References params!
```

#### 3. üìù Static Answers: No Quotes
```json
// ‚ùå BAD
{
  "params": {},
  "answer": "'x ‚â§ ‚àí4'"  // Shows quotes in UI!
}

// ‚úÖ GOOD
{
  "params": {},
  "answer": "x ‚â§ ‚àí4"  // Clean display
}
```

#### 4. ‚ûó Always Round Division
```json
// ‚ùå BAD
"answer": "(b - c) / a"  // Shows 0.6666666666

// ‚úÖ GOOD
"answer": "round((b - c) / a, 2)"  // Shows 0.67
```

#### 5. üî¢ Phase Numbering: Avoid .10
```json
// ‚ùå BAD
"phase": 10.10  // Equals 10.1 in JavaScript!

// ‚úÖ GOOD
"phase": 10.11  // Use 10.11, 10.12, 10.13, etc.
```
